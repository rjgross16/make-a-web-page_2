Why am I seeing this?

You are on a browser (hopefully, or else then I have no answer for you). You told your browser to go to this page, and so your computer, since it's dumb and has no mind of its own (yet), went ahead and texted its friend Web and said "Hey buddy, can you send me the files that my owner requested?"

Web said, "no problem, just give me a few nanoseconds," then retreived the files from a server via something called HTTP and sent them to your computer. Then your browser, since it's also dumb and has no mind of its own, interpeted the files that came through and displayed them on your screen.

Ok, but how did you make it?

I used a language called HTML, which stands for Hypertext Markup Language. HTML is made up of:

  • Text content - which is what you are seeing now
  • Markup - which is what this page looks like
  • References to other documents - like the logo at the top of the screen/li>
  • Links to other pages - like this link to a puppy video

How did you make it look like this?

Everything you see on the screen is part of an HTML element. You add these elements to your page using things called tags, and you can apply attributes, or styles, to those elements. So, for example, I added a link to the puppy video using a link tag (which looks like this: <a>), and I wrote in my code that all link elements should be my favorite shade of red and have bold font.

Oh, so how do your write HTML?

To start, you can learn one easy thing, which is that every HTML document has the same basic structure:

  • HTML <html> - which tells the computer that you are writing a document using HTML
  • Head <head> - where you usually put the styling of your page
  • Body <body> - where pretty much everything else goes

So now you're saying it is easy?

Well, to start is easy. But then what? You need to know some more essentials. For instance, there are other languages besides HTML. One language that's used for pretty much every web site is CSS. HTML controls the "structure" of a webpage, while CSS controls the "style" of the page.

You can put your styling directly in to your HTML, but CSS is essential to code writers because it helps them avoid a lot of repetition. I already gave an example of this above. To avoid having to tell my HTML every time that I wanted my links red, I instead only had to type it once in my CSS.

Tell me more about HTML and CSS

As I said, the <head> section is usually where you put styling, so if you have a CSS file you can simply put a link to it in the <head> section of your HTML and voila, your HTML will now listen to what your CSS tells it to do.

Programmers use a "tree-like structure" when coding their sites. We call it that because HTML elements can have other elements inside of them, kind of like a family tree. And just like a family, the "child" elements can "inherit" attributes from a "parent", which, again, helps programmers avoid repetition.

But what about these boxes?

You're catching on already! You can see that there are boxes on this page. But even when you can't see boxes on the screen, know that the page was still built using boxes. Every page has boxes! Every box on your screen is built of four elements:

  • Content - in the middle, contains the text and images you see on the screen. The other layers are like protection to the content.
  • Padding - this clears an area around the content. Like how there is a little space between the red text and the gray side of your screen.
  • Border - this goes around the content and padding. These boxes have a gray border on the bottom.
  • Margin - the outermost layer, it provides spaces between the boxes on the page.

How do you get the boxes where you want them to go?

Box sizing and positioning is one of the hardest parts of building a web page. You can set the size of your box with pixels, or in percentages. I used percentages on this page.
